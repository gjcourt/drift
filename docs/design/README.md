# design/

What we're building and **why** — proposals, RFCs, in-flight or recently shipped designs.

**Put here:**
- Proposals for new features, refactors, or architectural changes.
- Design records that capture the decision and the alternatives considered.
- Recently shipped designs that document the intent behind code that exists.

**Do not put here:**
- How the system is built today — `architecture/`.
- How to execute a phased rollout — `plans/`.
- Spike output / vendor evaluation — `research/`.

**Naming convention:** `<yyyy-mm-dd>-<topic>.md`
Examples: `2026-06-10-streaming-results-api.md`.

**Allowed `status:` values:** `Proposed`, `Accepted`, `Rejected`, `Implemented`, `Superseded`.

The doc stays put through state transitions — only `status:` and `superseded_by:` change.
